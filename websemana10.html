Activación Rápida: Conceptos Previos

En clases anteriores trabajamos con entidades tipo Vehicle para transporte.

Pregunta de Análisis:

¿Qué diferencia fundamental aporta un Worker frente a un Vehicle convencional en Simio?

¡Exacto!

A diferencia del Vehicle, el Worker puede:

✓ Ejecutar procesos en servidores (capacidad de proceso).
✓ Seguir turnos específicos (Work Schedules).
✓ Caminar por nodos sin necesidad de rutas físicas estrictas.

Worker: Operador Multifuncional Inteligente

El objeto Worker resuelve restricciones humanas en modelos de eventos discretos.

  • Recurso de Proceso: Requerido por un Server para funcionar.
  • Agente de Transporte: Mueve entidades entre estaciones.
  • Gestión de Descansos: Integración directa con matrices de turnos.

Tally Statistic: Más allá del reporte default

Simio entrega estadísticas estándar, pero a veces necesitamos KPIs específicos del negocio.

1

Medición Discreta

Registra valores puntuales (ej: tiempo de ciclo de una ruta específica).

2

Personalización

Permite analizar sub-procesos que los reportes globales ocultan.

[Imagen: Gráfico de barras de desempeño con una lupa enfocando un dato específico, representando la captura de estadísticas precisas. Paleta corporativa.]

Cómo implementar un Tally en 3 pasos

Integración en la lógica de procesos (Add-On Processes).

  • Paso 1: Crear el elemento TallyStatistic en la pestaña Definitions > Elements.
  • Paso 2: En la pestaña Processes, insertar el paso Tally en el flujo lógico.
  • Paso 3: Configurar el Value Expression (ej. tiempo actual menos tiempo de creación).
Value Expression: TimeNow - ModelEntity.TimeCreated
Tally Statistic Name: Tally_TiempoCiclo

Activación Analítica: Midiendo la Espera

Imagina un sistema logístico donde el material llega rápido, pero el Worker tarda en buscarlo.

Situación:

Si necesitamos medir exactamente el tiempo que una entidad espera por la llegada del Worker al servidor, ¿cómo configuramos el Tally?

Solución:

1. Usamos un Add-On Process en el Server activado por "Processing" (cuando el Worker ya llegó y empieza a trabajar).

2. El Value Expression del Tally sería la diferencia entre el tiempo actual y el tiempo en que la entidad entró al servidor (guardado previamente con un paso Assign).
